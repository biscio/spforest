\section{The data used for the examples}

This vignettes illustrate how to use spatial random forests to estimate the 
intensity of a point process. 
To that end, we will work with the data set \texttt{bei} from \spst, 
together with the covariates from the tropical forest data set. 
The covariates are loaded in the the package \pkg{sandbox} 
in the list \texttt{beisoil}.


\includegraphics{Intro-003}



Each element of the list \texttt{beisoil} is a \texttt{im} object from the package 
\pkg{spatstat-geom}. Roughly, this can be seen as a pixel images. 
In order to be able to work with this data, we need to transform them 
to have the same resolution. 
This have been done with the following code whose output is available in \texttt{beisoilres},

\begin{Schunk}
\begin{Sinput}
> zsoil <- NULL
> zsoil[[1]] <- beisoil[[1]]
> zsoil[[2]] <- beisoil[[2]]
> for (i in 3:15) {
+   W <- commonGrid(beisoil[[1]], beisoil[[i]])
+   zsoil[[i]] <- beisoil[[i]][W, drop = F]
+   rm(W)
+ }
> names(zsoil) <- c(
+   "elev", "grad", "Al", "B", "Ca",
+   "Cu", "Fe", "K", "Mg", "Mn",
+   "P", "Zn", "N", "N_min", "pH"
+ )
\end{Sinput}
\end{Schunk}

\section{Fitting a spatial intensity tree}

The function \texttt{treerec} allows to fit a spatial intensity tree 
to a point pattern. 

\begin{Schunk}
\begin{Sinput}
> inttree <- treerec(
+   X = spatstat.data::bei,
+   score = "lcv2",
+   mtry = 1,
+   minpts = 500,
+   listcovariates = beisoilres
+ )
\end{Sinput}
\end{Schunk}

The output is an object of class \texttt{sptree} define in the package \pkg{Rsandbox} 
together with methods to print, plot, summarise and predict the intensity in a given region.

The function \texttt{summarise} displayed the intensity tree 
as a data.frame where each row represent a leave of the tree.

\section{Fitting a spatial intensity forest}

The function \texttt{RforestPP} compute a random intensity forest. 
The function allows to parallelise the computation by allocating 
each tree intensity tree computation to a core, via use of \texttt{mclapply}. 

\begin{Schunk}
\begin{Sinput}
> forest <- RforestPP(
+   X = spatstat.data::bei,
+   listcovariates = beisoilres,
+   score = "lcv2",
+   p = 0,
+   Ntree = 3,
+   mtry = 1 / 3,
+   minpts = 500,
+   cores_trees = 1
+ )
\end{Sinput}
\end{Schunk}

The output is an object of the class \texttt{spforest} define in the package \pkg{Rsandbox} together a set of methods defined below.

\subsection{Standard methods on spforest object}

The \texttt{print} function returns the main information on the forest intensity estimate.
\begin{Schunk}
\begin{Sinput}
> print(forest)
\end{Sinput}
\begin{Soutput}
Intensity forest estimate of point patterns with 3604 points.

15 covariables used, with names: elev, grad, Al, B, Ca, Cu, Fe, K, Mg, Mn, P, Zn, N, N_min, pH. 
Spatial intensity forest with 3 trees.
\end{Soutput}
\end{Schunk}

The \texttt{plot} function of an \texttt{spforest} object 
return an \texttt{im} obect from \spst and plot it. 
\begin{Schunk}
\begin{Sinput}
> plot(forest)
\end{Sinput}
\end{Schunk}
\includegraphics{Intro-014}

Prediction in new area can be obtained via \texttt{predict} which 
accept as input a xy-vector for one point, or a \texttt{ppp} object.

\begin{Schunk}
\begin{Sinput}

\begin{Soutput}

\subsection{Working with Random intensity forest}

One of the main interest of random forest is the ability to compute 
a variable importance and an Out-Of-Bag (OOB) Score. 

The OOB score is return by the function \texttt{OOBscr.spforest}.
\begin{Schunk}
\begin{Sinput}
> OOBscr.spforest(forest = forest, cores = 1)
\end{Sinput}
\begin{Soutput}
[1] -12668.53
\end{Soutput}
\end{Schunk}

The importance score of each variables can be computed on each tree 
of the forest with the function \texttt{importance.spforest}. 
The second argument is the position of the covariates 
we are analysing in the argument \texttt{listcovariates} of \texttt{RforestPP}.
\begin{Schunk}
\begin{Sinput}
> importance.spforest(forest = forest, id_cov=3)
\end{Sinput}
\begin{Soutput}
[1] 0 0 0
\end{Soutput}
\end{Schunk}

To analyse how the VIP varies accross the trees, we can plot 
use a boxplot. 
\begin{Schunk}
\begin{Sinput}
> boxplot_spforest(forest)
\end{Sinput}
\end{Schunk}
\includegraphics{Intro-019}

However, it is more traditional in random forest to take the mean of the 
VIP of all the trees which is made in the function \texttt{vipplot.spforest}.
\begin{Schunk}
\begin{Sinput}
> vipplot.spforest(forest)
\end{Sinput}
\end{Schunk}
\includegraphics{Intro-020}

It is also possible to immediately sort the VIP in decreasing order. Note however than when the number of trees is too small, there can be some variation between the two run of the function \texttt{vipplot.spforest}.
\begin{Schunk}
\begin{Sinput}
> vipplot.spforest(forest, sorted = T)
\end{Sinput}
\end{Schunk}
\includegraphics{Intro-021}
